Theory & Model Notes (Traceable)

1) Contact Ratio (ŒµŒ±) ‚Äî two equivalent checks

The tool evaluates the transverse contact ratio by two independent expressions, reported side-by-side. The first uses the extracted working Line of Action (LOA) length; the second uses the equivalent effective contact length derived from pitch-circle arcs.

Definitions (transverse plane): t0 = œÄ m (circular pitch) pb = t0 cos Œ± (base pitch) tg = t0 cos Œ± (projected pitch on LOA) [same numeric value as pb in this implementation] Check A (LOA-based): ŒµŒ± = L_LOA / tg Check B (arc-based): ea = arc(B‚ÜíP on pinion pitch circle) + arc(P‚ÜíC on gear pitch circle) ŒµŒ± = ea / t0

LOA is extracted by computing the raw contact path from the rack-generation kinematics and retaining only the points that are simultaneously inside both addendum circles (pinion and gear). The longest contiguous segment of that feasible region is the working LOA.

2) Involute generation and LOA extraction (rack method)

A straight rack flank is sampled and mapped to the pinion frame using the classic rolling constraint. The result gives both: (i) a working involute flank, and (ii) a raw contact path which is filtered to obtain the working LOA.

Rack line (sampled): y_r ‚àà [ -C_f m, +C_k m ] x_r = -tan(Œ±) ¬∑ y_r Kinematic mapping (pinion pitch radius r_o1): Œ∏ = K / r_o1 K = -( y_r ¬∑ dy/dx + x_r ) Contact path in pitch-point frame: x_cp = x_r + K y_cp = y_r Working LOA: keep points where: || (x_cp, y_cp + r_o1) || ‚â§ r_a1 and || (x_cp, y_cp - r_o2) || ‚â§ r_a2 (inside both addendum circles) choose the longest contiguous run ‚Üí LOA

3) Trochoid / root completion (rack fillet)

The root region is completed by constructing a rack fillet arc (target radius rc) and mapping it through the same rolling transformation, generating the trochoid portion of the tooth boundary. The code includes robust radius targeting (bisection + secant fallback) to match rc.
